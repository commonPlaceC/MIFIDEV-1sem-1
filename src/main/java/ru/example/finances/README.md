# Система Управления Личными Финансами

Консольное приложение для управления личными финансами

## Функциональность

### Основные возможности
- **Управление доходами и расходами** - добавление, просмотр и категоризация транзакций
- **Система бюджетирования** - установка лимитов по категориям с отслеживанием превышений
- **Многопользовательская система** - регистрация, авторизация и изолированные данные пользователей
- **Финансовые отчеты** - детальная статистика по доходам, расходам и бюджетам
- **Система уведомлений** - предупреждения о превышении бюджетов и перерасходах
- **Персистентность данных** - сохранение в JSON файлах с автоматическим резервным копированием

## Запуск

### 1. Сборка проекта
```bash
./gradlew build
```

### 2. Запуск приложения
```bash
./gradlew run
```

## Руководство пользователя

### Первый запуск

1. При первом запуске выберите "Зарегистрироваться"
2. Введите имя пользователя (3-20 символов, буквы, цифры, подчеркивание)
3. Создайте пароль (минимум 6 символов)
4. Укажите email адрес
5. После регистрации войдите в систему

### Основные операции

#### Управление финансами
- **Добавить доход**: Меню → Управление финансами → Добавить доход
- **Добавить расход**: Меню → Управление финансами → Добавить расход
- **Просмотр транзакций**: Показывает последние 10 операций

#### Управление бюджетами
- **Установить бюджет**: Укажите категорию и лимит расходов
- **Мониторинг**: Система автоматически отслеживает превышения
- **Статистика**: Детальная информация по всем бюджетам

#### Переводы между пользователями
- **Отправка перевода**: Укажите получателя, сумму и описание
- **Получение**: Переводы обрабатываются автоматически при входе
- **Отмена**: Возможность отменить необработанные переводы

### Структура данных

Приложение создает следующие файлы:
```
data/
├── users.json              # Данные пользователей
├── pending_transfers.json  # Ожидающие переводы
└── wallets/
    ├── username1.json      # Кошелек пользователя 1
    └── username2.json      # Кошелек пользователя 2
```

## Конфигурация

### Настройки валидации
- Имя пользователя: 3-20 символов (буквы, цифры, _)
- Пароль: минимум 6 символов
- Суммы: положительные числа
- Категории: 1-50 символов (буквы, цифры, пробелы, _, -)

### Уведомления
- **Предупреждение**: при использовании 80% бюджета
- **Критическое**: при использовании 90% бюджета
- **Превышение**: при превышении лимита бюджета
- **Перерасход**: когда расходы превышают доходы
